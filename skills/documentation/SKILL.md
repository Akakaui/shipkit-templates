# DOCUMENTATION — README, API Docs, Architecture Docs

## PURPOSE

Ensures comprehensive documentation for every project.

## TRIGGERS

- "Document my project"
- "Create README"
- "Write API docs"

## REQUIRED DOCUMENTATION

### README.md

- Project purpose
- Classification & scope
- Folder/module structure
- Architecture overview
- State management
- Auth & security
- Database & API design
- Testing strategy
- Deployment / CI/CD
- Monitoring & logging
- Known limitations
- Handoff instructions

### Architecture Docs

- Component hierarchy
- Data flow diagrams
- API design
- Database schema
- Authentication flow
- Error handling strategy

### API Documentation

- All endpoints documented
- Request/response examples
- Authentication requirements
- Rate limits
- Error codes

## RULES

- ALWAYS create README
- ALWAYS document API endpoints
- ALWAYS document complex logic
- ALWAYS verify setup instructions
- NEVER skip documentation
- NEVER leave outdated docs
